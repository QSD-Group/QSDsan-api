# FastAPI Migration Guide

## Status: Complete

The Flask → FastAPI migration is done. The Flask layer (`app/blueprints/`, `wsgi.py`, `app/__init__.py` factory) has been removed. `app/main.py` is the sole entry point.

---

## What was migrated

| Component | Flask (removed) | FastAPI (current) |
|-----------|-----------------|-------------------|
| App entry point | `wsgi.py` → `app/__init__.py` factory | `app/main.py` |
| Routing | `app/blueprints/` (Blueprint handlers) | `app/routers/` (APIRouter handlers) |
| Validation | Manual `request.args.get()` + if-checks | Pydantic + FastAPI `Query()` params |
| Documentation | Flasgger decorators | Auto-generated OpenAPI at `/docs`, `/redoc` |
| Error handling | `make_response(jsonify(...))` | `HTTPException` + middleware |
| CORS | `flask-cors` | `fastapi.middleware.cors.CORSMiddleware` |

---

## Architecture

### Entry point: `app/main.py`
- Registers routers under `/api/v1/`
- Middleware stack (order: SecurityHeaders → RateLimit → Performance → ErrorHandler)
- CORS configured here — **lock down `allow_origins` before public deployment**

### Routers: `app/routers/`
- `htl.py` — `/htl/calc`, `/htl/county`
- `combustion.py` — `/combustion/calc`, `/combustion/county`
- `fermentation.py` — `/fermentation/calc`, `/fermentation/county`
- `health.py` — `/health`, `/ready`, `/metrics`, `/performance`
- `v2_example.py` — placeholder for future v2 endpoints

### Models: `app/models/`
Pydantic v2 request/response models. Enums define valid units and waste types.

### Services: `app/services/`
All scientific computation lives here. Routers are thin wrappers.

- `htl_service.py` — uses `exposan.htl` + `chaospy`. `/calc` runs live simulation (seconds). `/county` reads pre-computed CSV.
- `combustion_service.py` — uses `biosteam`/`biorefineries`. Same pattern.
- `fermentation_service.py` — same pattern.

### Data: `app/data/`
Pre-computed CSV files for county lookups. Updated by one-time scripts in `scripts/`.

- `htl/htl_data.csv` — includes `MDSP ($/gal)` and `GWP (lb CO2/gal)` columns (generated by `scripts/precompute_htl.py`)
- `combustion/combustion_data.csv` — includes per-waste-type result columns for all 21 NJ counties
- `fermentation/fermentation_data.csv` — includes `Annual Ethanol`, `Price`, `GWP` columns

### Middleware: `app/middleware/`
- `security.py` — `RateLimitMiddleware` (in-process, per-IP), `SecurityHeadersMiddleware`
- `performance.py` — `PerformanceMiddleware` (request timing, slow request tracking)
- `error_handler.py` — `ErrorHandlerMiddleware` (catches unhandled exceptions)

## Running

```bash
# FastAPI (production target)
uv run uvicorn app.main:app --reload --port 5000

# Docker
docker build -t waste-energy-api .
docker run -p 5000:5000 waste-energy-api
```

## Remaining work

- **CORS**: Change `allow_origins=["*"]` in `app/main.py` to the actual frontend domain before public deployment
- **Config**: Rate limit values and other settings are hardcoded in `app/main.py` — extract to env vars / `config.py` if needed
- **Rate limiting**: Current implementation is in-process (resets on restart, not shared across multiple containers). Replace with Redis-backed limiting if horizontal scaling is needed
- **`/calc` latency**: Live scientific simulations take seconds. Acceptable for research use; add async task queue (Celery/ARQ) if low-latency is ever required
- **Anaerobic digestion**: Not yet implemented (no router, service, or data)
